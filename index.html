What are Build Tools?

Build tools are programs that automate the creation of executable applications from source code. Building incorporates compiling, linking and packaging the code into a usable or executable form.

In English:

Build tools do all the things you don't like to do to make your code glorious!

Like what?

ask if anyone can name things they hate doing

Auto-prefix your css

Write this:


.c-test {
  display: flex;
}

Not this:


.c-test{
  display: -webkit-box;
  display: -webkit-flex;
  display: -ms-flexbox;
  display: flex
}

Minify Code

Write css, js, html, (even images) as normal, build will minify for you

Combine Code

Combine all the 3rd party libraries with your own code into 1 file

Generate/inject Code

  • Config files
  • Custom icon libraries
  • Custom Replacements
  • Anything!

Develop Faster

  • Watch for changes
  • Update server automatically
  • Even test many devices automatically!

Testing

6. Build Your Project

Add a new task in gulpfile.js:


gulp.task('build', () => {
  return gulp
    .src('app/**/*')
    .pipe(gulp.dest('dist/'));
})

src - where to look (file glob)

dest - where to build (specific folder, follows same pattern as src unless otherwise specified)

Make sure it's really copying!

Create file app/test.css


.test {
  display: flex;
}

Now in console:

$ gulp build

Should see gulp running a build

Check your dist/ folder

Do you see your test.css file?

7. Watch your code

Because running $ gulp build every time is a pain

Add a new task gulpfile.js:


gulp.task('watch', ['build'], () => {

what task to run when a change happens

8. Autoprefix your CSS

because this is a lame build process without magic

In console, install autoprefixer

npm install gulp-autoprefixer --save-dev

In gulpfile.js,

add autoprefixer package at the top:


const gulp = require('gulp')
const autoprefixer = require('gulp-autoprefixer')

In gulpfile.js,

update build to include autoprefixer


gulp.task('build', () => {
  return gulp
    .src('app/**/*')
    .pipe(autoprefixer({ browsers: ['last 2 versions', '> 5%'] }))
    .pipe(gulp.dest('dist/'));
})

Make sure it's autoprefixing!

Run $ gulp build

Check dist/test.css

You should see something like:


.test {
  display: -webkit-box;
  display: -ms-flexbox;
  display: flex;
  color: red;
}

9. Subtasks

why is this build awful?

answer: because it only builds css... anything else will break it.

Break Tasks down by file type

a : denotes a subtask

Make CSS have Subtasks

In gulpfile.js, make our tasks specific to CSS


// css
gulp.task('css:build', () => {
  return gulp
    .src('app/**/*.css')
    .pipe(autoprefixer({ browsers: ['last 2 versions', '> 5%'] }))
    .pipe(gulp.dest('dist/'));
})

gulp.task('css:watch', ['css:build']() => {
  gulp.watch(`app/**/*.css`, ['css:build']);
})

Make HTML Tasks

In gulpfile.js, copy the CSS tasks, and use for HTML


// html
gulp.task('html:build', () => {
  return gulp
    .src('app/**/*.html')
    .pipe(gulp.dest('dist/'));
})

gulp.task('html:watch', ['html:build'], () => {
  gulp.watch(`app/**/*.html`, ['html:build']);
})

Make JS Tasks

In gulpfile.js, copy the HTML tasks, and use for JS


// js
gulp.task('js:build', () => {
  return gulp
    .src('app/**/*.js')
    .pipe(gulp.dest('dist/'));
})

gulp.task('js:watch', ['js:build'], () => {
  gulp.watch(`app/**/*.js`, ['js:build']);
})

Test your changes

add app/test.js, app/test.html to your site.

Empty your dist/ directory

Run a task to see it build

It should only build the thing specified


$ gulp css:build
$ gulp css:watch
$ gulp js:build
$ gulp js:watch
$ gulp html:build
$ gulp html:watch

10. Concurrent Tasks

combine similar tasks so they can run together

Make one build and watch Task

in gulpfile.js:


gulp.task('build', ['css:build', 'js:build', 'html:build'])
gulp.task('watch', ['css:watch', 'js:watch', 'html:watch'])

the 2nd argument means these tasks can be run concurrently, not sequentially.

Test your changes

$ gulp build

Should run all builds separately

gulp.task('FILETYPE:clean', () => {

const minimist  = require('minimist')
const options = minimist(process.argv.slice(2), {

    production: false
  }
});

In gulpfile.js:


console.log('is production?', options.production);

Now, run $ gulp build --production

Console should log true

13. Use your production flag

to only minify code for production builds

Install gulp-uglify and gulp-util

$ npm install gulp-uglify --save-dev

$ npm install gulp-util --save-dev

Add them as requirements

Near top of gulpfile.js


const gutil = require('gulp-util')
const uglify = require('gulp-uglify')

Add to your JS Build


gulp.task('js:build', () => {
  return gulp
    .src('app/**/*.js')
    .pipe(options.production ? uglify() : gutil.noop())
    .pipe(gulp.dest('dist/'));
})

Test it!

$ gulp build --production

dist/test.js should be minified

$ gulp build

dist/test.js should be expanded

This is awesome, because it means we don't have to waste time doing lame things we don't need when watching for changes

Install Connect & Run-Sequence

  2. Go to http://localhost:8080/dist/test.html in browser
  3. See your beautiful site!

Add serve to the watch task

Update the serve task


gulp.task('serve', () => {
  connect.server({
    livereload: true
  })
})

Update your :build tasks

Add a .pipe() to the last line of each build, like so:


gulp.task('TASKNAME:build', () => {
  return gulp
    .src('app/**/*.TASKNAME')
    .pipe(gulp.dest('dist/'))
    .pipe(connect.reload());
})

Update the watch task


gulp.task('watch', function(cb){
  return runSequence(
    'serve',
    ['css:watch', 'js:watch', 'html:watch']
  )
})
"callback" allows it to return correctly

Test your watch

  1. $ gulp watch
  2. Go to http://localhost:8080/dist/test.html in browser
  3. Change a file in app/
  4. Watch it magically reload!
